git

commands and definitions

branch
see local branches
fetch
updates .git/refs/remotes/origin/master, a file containing information about the remote repo.
diff
after fetch, diffs local repo against remote.

how to

basic workflow

  1. configure with config
  2. clone a repository 
    git clone https://github.com/cornellier/johnsdocs.git
cd johnsdocs
  3. Create and/or edit a file. See changes: git diff foo.txt
  4. git add foo.txt (stage) the file to the index. Undo with git reset HEAD foo.txt. See changes between staged file and remote repo git diff --cached foo.txt
  5. Do more changes. The file now has three states in three different places:
    • HEAD
    • staged
    • the working directory
    Abandon the changes in the working directory and reset it to the state of the stage: git checkout foo.txt.
  6. git commit foo.txt
  7. Push to HEAD: git push

config

git config --global user.name "John Cornellier"
git config --global user.email john@johnsdocs.com
git config --global alias.a add
git config --global alias.co commit
git config --global alias.st status

See settings with git config --list.

create and add a file

echo .DS_Store >> .gitignore
git add .gitignore

see staged (added) non-commited changes

git diff --cached

diff with origin

git fetch origin
git diff origin -- [local-path]

show files in local directory, not in HEAD

git ls-files --others --exclude-standard

undo add before commit

git reset foo.txt

ammend the last commit message

git commit --amend

undo the last commit

git commit -m "oops"
git reset HEAD~

remove local uncommited changes

tracked unstaged: modified but not added: git checkout foo.txt

tracked staged: have been added, aka moved to staging aka added to index: git reset --hard

untracked: new files, by definition unstaged, remove (no undo): git clean -f. To remove directories add -d.
